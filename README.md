## Тестовый стенд

### Состав стенда

Тестовый стенд реализован в формате Node.js-проекта и включает три тестируемых приложения, идентичных по содержанию, но различающихся технологией формирования итогового HTML:

| Приложение       | Технология            | Порт | Описание                                                              |
| ---------------- | --------------------- | ---- | --------------------------------------------------------------------- |
| `Native_HTML`    | Static Rendering      | 8080 | Эталонная страница, все элементы присутствуют в исходном коде         |
| `Native_Angular` | Client Side Rendering | 4300 | DOM формируется в браузере после выполнения JavaScript-бандла         |
| `Angular_SSR`    | Server Side Rendering | 4200 | HTML генерируется на сервере, затем выполняется гидратация на клиенте |

Все три приложения содержат идентичный набор из 53 нарушений WCAG 2.1 уровней A и AA.

### Инструменты проверки

| Инструмент | Версия | Браузер                   | Проверяемые стандарты              |
| ---------- | ------ | ------------------------- | ---------------------------------- |
| axe-core   | 4.11.3 | Chromium, Firefox, WebKit | WCAG 2.0 A, AA; WCAG 2.1 A, AA     |
| Pa11y      | 8.0.0  | Chromium                  | WCAG 2.0 AA                        |
| Lighthouse | 12.0.0 | Chromium                  | WCAG 2.1 (категория accessibility) |

### Фреймворк оркестрации

Playwright 1.60.0 с тремя проектами (Chromium, Firefox, WebKit). Последовательное выполнение тестов (`workers: 1`), максимальное время теста — 60 секунд.

### Управление серверами

Автоматический запуск и остановка серверов приложений через `child_process.spawn` с опросом готовности HTTP-эндпоинта (интервал 1 с, таймаут 60 с). Принудительное освобождение портов после завершения каждого тестового набора.

### Собираемые метрики

- Время выполнения инструмента (мс)
- Количество обнаруженных нарушений
- Наличие указания на DOM-элемент, содержащий нарушение
- Полный перечень нарушений в формате JSON
